Etiquetas para la creación de formularios (solo formato, no envío de datos)

Los formularios en HTML son la principal manera de recopilar información de los usuarios. La etiqueta <form> </form> es el contenedor básico de un formulario, dentro del cual se agrupan todas las etiquetas que permiten la interacción del usuario, como campos de texto, botones y opciones de selección.

Estas etiquetas son esenciales para crear formularios funcionales y accesibles, que permitan a los usuarios interactuar de manera efectiva con una página web.

Etiqueta <form>

<form> </form>

Esta es la etiqueta de apertura y cierre de un formulario en HTML. Todo el contenido del formulario, incluyendo campos de entrada, botones y otros elementos interactivos, debe estar contenido dentro de estas etiquetas.

Se utiliza para definir un formulario en la página web y especificar cómo y dónde se enviarán los datos del formulario, a través de atributos como action (la URL a la que se envían los datos) y method (el método de envío, como GET o POST).

Atributos

  • action: Define la URL a la que se envían los datos del formulario cuando el usuario lo envía.
  • method: Especifica el método HTTP que se utiliza al enviar los datos del formulario. Los valores comunes son GET y POST.
  • enctype: Define el tipo de codificación de los datos cuando se envían al servidor. Los valores comunes son application/x-www-form-urlencoded, multipart/form-data, y text/plain.
  • target: Indica dónde se muestra la respuesta que se recibe después de enviar el formulario. Los valores pueden ser:
    • _self: Muestra la respuesta en la misma ventana o pestaña.
    • _blank: Muestra la respuesta en una nueva ventana o pestaña.
    • _parent: Muestra la respuesta en el marco padre del formulario.
    • _top: Muestra la respuesta en la ventana o pestaña completa.
    • nombre de un iframe: Muestra la respuesta en un iframe específico dentro de la página.
  • autocomplete: Indica si el navegador debe completar automáticamente los valores del formulario. Puede ser on o off.
  • novalidate: Si está presente, indica que el formulario no debe ser validado al enviarse.
  • name: Asigna un nombre al formulario, que puede ser utilizado en scripts o estilos CSS.
  • rel: Define la relación entre el documento actual y la URL especificada en el atributo action. Común en formularios que usan target="_blank".
  • accept-charset: Especifica el conjunto de caracteres que el servidor acepta para el formulario. Ejemplos: UTF-8, ISO-8859-1.
  • class: Asigna una o más clases al elemento, permitiendo aplicar estilos CSS específicos.
  • id: Asigna un identificador único al formulario, útil para referencia en CSS o JavaScript.
  • style: Permite aplicar estilos CSS en línea directamente al elemento.
  • title: Proporciona un texto emergente (tooltip) que aparece cuando se pasa el cursor sobre el formulario.
  • lang: Especifica el idioma del contenido del elemento, ayudando a los navegadores y herramientas de accesibilidad a interpretar el texto correctamente.
  • dir: Define la dirección del texto dentro del elemento, que puede ser:
    • ltr: De izquierda a derecha.
    • rtl: De derecha a izquierda.
  • aria-*: Atributos ARIA (Accessible Rich Internet Applications) que mejoran la accesibilidad para usuarios con discapacidades. Ejemplos incluyen aria-label, aria-labelledby, aria-describedby, etc.

Ejemplo



Línea 9: "form action="/submit-form": El atributo action especifica que los datos del formulario se enviarán a la URL /submit-form cuando el usuario lo envíe.
Línea 10: method="POST": El atributo method define que el formulario enviará los datos usando el método POST, lo que es adecuado para enviar datos sensibles o grandes.
Línea 11: enctype="multipart/form-data": El atributo enctype establece que los datos del formulario se codificarán como multipart/form-data, lo cual es necesario para cargar archivos.
Línea 12: target="_blank": El atributo target indica que la respuesta del servidor se abrirá en una nueva pestaña o ventana del navegador.
Línea 13: autocomplete="on": El atributo autocomplete permite que el navegador complete automáticamente los campos del formulario con valores previamente ingresados por el usuario.
Línea 14: name="userForm": El atributo name asigna un nombre al formulario, lo que puede ser útil para referenciarlo en scripts o para identificarlo en los datos enviados.
Línea 15: id="userForm": El atributo id asigna un identificador único al formulario, permitiendo referenciarlo en CSS o JavaScript.



Etiqueta <legend>

<legend> </legend>

La etiqueta "legend" se utiliza en conjunto con "fieldset" para proporcionar un título o descripción a un grupo de elementos del formulario. Este título aparece dentro del recuadro creado por "fieldset", generalmente en la esquina superior izquierda.

Se coloca inmediatamente después de la apertura de "fieldset" para describir el propósito del grupo de elementos contenidos en él.

Atributos

  • align:
    • Descripción: Define la alineación del texto dentro del <legend>.
    • Valores posibles: left, right, center. (Nota: Este atributo está en desuso en HTML5; se recomienda usar CSS para la alineación.)
  • class:
    • Descripción: Permite asignar una o más clases al elemento para aplicar estilos CSS específicos.
    • Valores posibles: Cualquier nombre de clase válido.
  • id:
    • Descripción: Proporciona un identificador único para el elemento, que puede ser utilizado en CSS o JavaScript para manipularlo o referenciarlo.
    • Valores posibles: Cualquier valor de identificación único válido.
  • style:
    • Descripción: Permite aplicar estilos CSS en línea directamente al elemento.
    • Valores posibles: Cualquier regla CSS válida.
  • title:
    • Descripción: Proporciona un texto emergente (tooltip) que aparece cuando se pasa el cursor sobre el elemento.
    • Valores posibles: Cualquier texto.
  • lang:
    • Descripción: Especifica el idioma del contenido del elemento, ayudando a los navegadores y herramientas de accesibilidad a interpretar el texto correctamente.
    • Valores posibles: Códigos de idioma como en, es, fr, etc.
  • dir:
    • Descripción: Define la dirección del texto dentro del elemento, que puede ser "ltr" (de izquierda a derecha) o "rtl" (de derecha a izquierda).
    • Valores posibles: ltr, rtl, auto.
  • aria-*:
    • Descripción: Atributos ARIA (Accessible Rich Internet Applications) que mejoran la accesibilidad para usuarios con discapacidades.
    • Valores posibles: Dependiendo del atributo ARIA específico, como aria-label, aria-labelledby, aria-describedby, etc.

Ejemplo


Línea 11: "legend id="formLegend" class="form-legend" style="color: blue;""Información Personal"/legend"
id="formLegend": Asigna un identificador único al elemento "legend", lo que permite referenciarlo en CSS o JavaScript.
class="form-legend": Aplica una clase CSS llamada form-legend al elemento "legend", lo que permite aplicar estilos definidos en una hoja de estilos externa.
style="color: blue;": Aplica un estilo en línea que establece el color del texto del "legend" a azul.
Información Personal: Es el texto que se muestra como título de la sección del formulario, enmarcado dentro de la etiqueta "fieldset"



Etiqueta <label>

<label> </label>

La etiqueta "label" se utiliza para asociar un texto descriptivo con un control de formulario (como un campo de entrada o una casilla de verificación). Esto mejora la accesibilidad, ya que los usuarios pueden hacer clic en el texto del "label" para activar el control asociado.
Se usa junto con el atributo for que coincide con el id del control correspondiente, vinculando así el texto del "label" al control.

Atributos

  • for:
    • Descripción: Asocia la etiqueta <label> con un elemento de formulario, como un <input>, <textarea>, <select>, etc. El valor del atributo for debe coincidir con el valor del atributo id del elemento asociado.
    • Valores posibles: Cualquier valor que coincida con un id de un elemento de formulario.
  • form:
    • Descripción: Especifica el id del formulario al que pertenece la etiqueta <label>. Este atributo se utiliza principalmente cuando la etiqueta <label> y su elemento de formulario asociado están fuera del elemento <form>.
    • Valores posibles: Cualquier valor de id que corresponda a un formulario en el documento.
  • class:
    • Descripción: Permite asignar una o más clases al elemento para aplicar estilos CSS específicos.
    • Valores posibles: Cualquier nombre de clase válido.
  • id:
    • Descripción: Proporciona un identificador único para el elemento <label>, que puede ser utilizado en CSS o JavaScript para manipularlo o referenciarlo.
    • Valores posibles: Cualquier valor de identificación único válido.
  • style:
    • Descripción: Permite aplicar estilos CSS en línea directamente al elemento <label>.
    • Valores posibles: Cualquier regla CSS válida.
  • title:
    • Descripción: Proporciona un texto emergente (tooltip) que aparece cuando se pasa el cursor sobre el elemento <label>.
    • Valores posibles: Cualquier texto.
  • lang:
    • Descripción: Especifica el idioma del contenido del elemento <label>, ayudando a los navegadores y herramientas de accesibilidad a interpretar el texto correctamente.
    • Valores posibles: Códigos de idioma como en, es, fr, etc.
  • dir:
    • Descripción: Define la dirección del texto dentro del elemento <label>, que puede ser "ltr" (de izquierda a derecha) o "rtl" (de derecha a izquierda).
    • Valores posibles: ltr, rtl, auto.
  • aria-*:
    • Descripción: Atributos ARIA (Accessible Rich Internet Applications) que mejoran la accesibilidad para usuarios con discapacidades.
    • Valores posibles: Dependiendo del atributo ARIA específico, como aria-label, aria-labelledby, aria-describedby, etc.

Ejemplo



Línea 12: "label for="name""Nombre:"/label": for="name": Asocia la etiqueta "label" con el elemento de formulario cuyo id es name. Esto ayuda a mejorar la accesibilidad y permite que al hacer clic en el texto de la etiqueta, el cursor se enfoque en el campo de entrada correspondiente.
Línea 15: "label for="email""Correo electrónico:"/label": for="email": Asocia la etiqueta "label" con el elemento de formulario cuyo id es email.
Línea 18: "label for="password""Contraseña:"/label": for="password": Asocia la etiqueta "label" con el elemento de formulario cuyo id es password.

Etiqueta <input>

<input>

Esta etiqueta es extremadamente versátil y se utiliza para crear una amplia variedad de campos de entrada en un formulario, como campos de texto, contraseñas, botones de radio, casillas de verificación, entre otros. Es una etiqueta auto-contenida, lo que significa que no requiere una etiqueta de cierre.
Se emplea con diferentes tipos especificados en el atributo type, como text, password, email, checkbox, radio, entre otros, para definir el tipo de campo de entrada.

Atributos

  • type:
    • Descripción: Especifica el tipo de control que se debe mostrar. Los valores pueden ser text, password, submit, radio, checkbox, email, number, date, etc.
    • Valores posibles: text, password, submit, radio, checkbox, email, number, date, file, hidden, reset, search, tel, url, etc.
  • name:
    • Descripción: Asigna un nombre al control de formulario, que se usa para identificar los datos del campo al enviarse al servidor.
    • Valores posibles: Cualquier nombre válido.
  • value:
    • Descripción: Define el valor del control. Para los campos de entrada de texto, este es el texto que aparece por defecto; para los botones, es el texto que se muestra en el botón.
    • Valores posibles: Cualquier texto.
  • id:
    • Descripción: Proporciona un identificador único para el control, útil para referencia en CSS o JavaScript.
    • Valores posibles: Cualquier identificador único válido.
  • placeholder:
    • Descripción: Muestra un texto de ejemplo en el campo de entrada cuando está vacío, que desaparece cuando el usuario empieza a escribir.
    • Valores posibles: Cualquier texto.
  • required:
    • Descripción: Indica que el campo es obligatorio y debe ser completado antes de enviar el formulario.
    • Valores posibles: Sin valor (presente o ausente).
  • readonly:
    • Descripción: Hace que el campo sea de solo lectura, evitando que el usuario modifique su contenido.
    • Valores posibles: Sin valor (presente o ausente).
  • disabled:
    • Descripción: Desactiva el campo, evitando que el usuario interactúe con él.
    • Valores posibles: Sin valor (presente o ausente).
  • maxlength:
    • Descripción: Establece el número máximo de caracteres que el usuario puede ingresar en un campo de texto.
    • Valores posibles: Un número entero.
  • min:
    • Descripción: Define el valor mínimo permitido para campos de tipo number, date, range, etc.
    • Valores posibles: Un valor que se alinee con el tipo del campo.
  • max:
    • Descripción: Define el valor máximo permitido para campos de tipo number, date, range, etc.
    • Valores posibles: Un valor que se alinee con el tipo del campo.
  • step:
    • Descripción: Define el intervalo entre los valores permitidos para campos de tipo number o range.
    • Valores posibles: Un valor numérico positivo.
  • pattern:
    • Descripción: Especifica una expresión regular que el valor del campo debe coincidir para ser válido (para campos de tipo text).
    • Valores posibles: Una expresión regular en formato de texto.
  • autocomplete:
    • Descripción: Indica si el navegador debe autocompletar el campo con valores previamente ingresados.
    • Valores posibles: on o off.
  • form:
    • Descripción: Especifica el id del formulario al que pertenece el control. Se utiliza si el campo de entrada está fuera del elemento <form>.
    • Valores posibles: Cualquier valor de id que corresponda a un formulario en el documento.
  • aria-*:
    • Descripción: Atributos ARIA que mejoran la accesibilidad para usuarios con discapacidades.
    • Valores posibles: Dependiendo del atributo ARIA específico, como aria-label, aria-labelledby, aria-describedby, etc.

Ejemplo


Línea 6: "input type="text" id="textInput" name="textInput" placeholder="Escribe aquí"" type="text": Define un campo de entrada de texto. placeholder="Escribe aquí": Muestra un texto de ejemplo en el campo cuando está vacío.

Línea 10: "input type="password" id="passwordInput" name="passwordInput" placeholder="Ingresa tu contraseña"" type="password": Define un campo de entrada para contraseñas, ocultando los caracteres ingresados.

Línea 14: "input type="submit" id="submitButton" value="Enviar"" type="submit": Define un botón que envía el formulario.

Línea 18: "input type="email" id="emailInput" name="emailInput" placeholder="example@example.com"" type="email": Define un campo de entrada para correos electrónicos y valida el formato del correo ingresado.

Línea 22: "input type="number" id="numberInput" name="numberInput" min="1" max="10""

  • type="number": Define un campo de entrada para números con validación de rango.
  • min="1" y max="10": Establecen el rango permitido para el valor.

Línea 26: "input type="date" id="dateInput" name="dateInput"" type="date": Define un campo de entrada para seleccionar fechas.

Línea 30: "input type="file" id="fileInput" name="fileInput"" type="file": Define un campo de entrada para seleccionar archivos.

Línea 34: "input type="hidden" name="hiddenField" value="valorOculto"" type="hidden": Define un campo oculto que no es visible para el usuario.

Línea 38: "input type="search" id="searchInput" name="searchInput" placeholder="Buscar..."" type="search": Define un campo de entrada para búsquedas, con características específicas para realizar búsquedas.

Línea 42: "input type="range" id="rangeInput" name="rangeInput" min="0" max="100" step="10""

  • type="range": Define un campo de entrada para seleccionar un valor dentro de un rango utilizando un control deslizante.
  • min="0", max="100", y step="10": Establecen el rango y el intervalo de los valores permitidos.

Etiqueta <button>

<button> </button>

Se utiliza para crear un botón interactivo en una página web. Los botones pueden ser utilizados para enviar formularios, ejecutar scripts de JavaScript, o realizar otras acciones en respuesta a eventos del usuario.

Atributos

  • type: Especifica el tipo de botón, como "button" para un botón genérico, "submit" para enviar un formulario o "reset" para restablecer el formulario.
  • id: Asigna un identificador único al `button`, útil para aplicar estilos específicos o manipular el botón con JavaScript.
  • class: Aplica una o más clases al `button`, permitiendo usar estilos CSS definidos o seleccionarlo con JavaScript.
  • disabled: Deshabilita el `button`, impidiendo que el usuario interactúe con él.
  • name: Asigna un nombre al `button`, que se envía junto con los datos del formulario si el botón es de tipo "submit".
  • value: Define el valor del `button`, que se envía con los datos del formulario si es de tipo "submit".
  • style: Permite aplicar estilos CSS en línea al `button`, personalizando su apariencia.
  • title: Proporciona un texto emergente (tooltip) que aparece cuando se pasa el cursor sobre el `button`, ofreciendo información adicional.
  • onclick: Especifica un código JavaScript que se ejecutará cuando se haga clic en el `button`.
  • autofocus: Hace que el `button` reciba automáticamente el foco cuando la página se carga.
  • form: Asocia el `button` a un formulario específico mediante su identificador.
  • formaction: Define la URL a la que se envían los datos del formulario cuando se hace clic en un `button` de tipo "submit".
  • formenctype: Especifica cómo se deben codificar los datos del formulario cuando se envían, aplicable solo a botones de tipo "submit".
  • formmethod: Define el método HTTP ("GET" o "POST") a utilizar al enviar un formulario, aplicable solo a botones de tipo "submit".
  • formnovalidate: Indica que no se debe validar el formulario al enviarlo, aplicable solo a botones de tipo "submit".
  • formtarget: Especifica dónde se debe mostrar la respuesta después de enviar el formulario, aplicable solo a botones de tipo "submit".

Ejemplo




Línea 1: id="botonEnviar" Asigna un identificador único 'botonEnviar' al botón.
Línea 12: class="btn btn-enviar" Aplica las clases 'btn' y 'btn-enviar' para estilos CSS.
Línea 13: type="submit" Define el botón como un botón de envío de formulario.
Línea 14: title="Enviar el formulario" Proporciona un *tooltip* que muestra "Enviar el formulario" cuando el usuario pasa el cursor sobre el botón.
Línea 15: style="background-color: #4CAF50; color: white; padding: 10px 20px; border-radius: 5px;" Aplica estilos en línea para personalizar la apariencia del botón.
Línea 16: disabled Deshabilita el botón, impidiendo que el usuario lo presione.
Línea 17: name="envio" Asigna el nombre 'envio' al botón.
Línea 18: value="enviar" Define el valor del botón como "enviar".
Línea 19: onclick="alert('Formulario enviado!')" Ejecuta un código JavaScript cuando se hace clic en el botón.
Línea 20: autofocus Hace que el botón reciba automáticamente el foco al cargar la página.



Etiqueta <select>

<select> </select>

Se utiliza para crear un menú desplegable que permite a los usuarios seleccionar una opción de una lista de opciones. Es comúnmente utilizada en formularios.

Atributos

  • id: Asigna un identificador único al `select`, útil para aplicar estilos específicos o manipular el elemento con JavaScript.
  • class: Aplica una o más clases al `select`, permitiendo aplicar estilos CSS o seleccionar el elemento con JavaScript.
  • name: Define el nombre del `select`, que se enviará junto con el valor seleccionado cuando se envíe el formulario.
  • size: Especifica la cantidad de opciones visibles en el `select` sin necesidad de desplazamiento.
  • multiple: Permite seleccionar más de una opción a la vez.
  • disabled: Deshabilita el `select`, impidiendo que el usuario lo utilice.
  • required: Indica que se debe seleccionar al menos una opción antes de enviar el formulario.
  • autofocus: Hace que el `select` reciba automáticamente el foco al cargar la página.
  • form: Asocia el `select` con un formulario específico si está fuera del `form` correspondiente.
  • data-*: Permite almacenar datos personalizados en el `select` para su uso en JavaScript.

Ejemplo


Línea 11: id="opciones" Asigna un identificador único 'opciones' al `select`, útil para aplicarle estilos específicos o manipularlo con JavaScript.
Línea 12: class="opciones-seleccion" Aplica la clase 'opciones-seleccion' al `select`, permitiendo aplicar estilos CSS.
Línea 13: name="opcion" Define el nombre 'opcion' para el `select`, que se enviará junto con el valor seleccionado cuando se envíe el formulario.
Línea 14: size="4" Muestra 4 opciones visibles sin necesidad de desplazamiento en el `select`.
Línea 15: multiple Permite seleccionar más de una opción a la vez en el `select`.
Línea 16: required Indica que se debe seleccionar al menos una opción antes de enviar el formulario que contiene el `select`.
Línea 17: disabled Deshabilita el `select`, impidiendo que el usuario lo utilice (puedes quitar este atributo si quieres permitir la selección).
Línea 18: autofocus Hace que el `select` reciba automáticamente el foco al cargar la página.
Línea 19: data-info="seleccion-opciones" Almacena un dato personalizado 'seleccion-opciones' en el `select`, accesible mediante JavaScript.

Etiqueta <option>

<option> </option>

Se utiliza para definir una opción dentro de un menú desplegable creado con la etiqueta select. Cada option representa una opción que el usuario puede seleccionar.

Atributos

  • value: Define el valor que se enviará al servidor cuando se seleccione esta opción.
  • selected: Preselecciona esta opción cuando se carga la página, mostrándola como la opción seleccionada por defecto.
  • disabled: Deshabilita la opción, impidiendo que el usuario la seleccione.
  • label: Proporciona un nombre alternativo que puede mostrarse en lugar del texto de la opción.
  • class: Asigna una o más clases a la opción, permitiendo aplicar estilos CSS específicos.
  • id: Asigna un identificador único a la opción, útil para manipularla con CSS o JavaScript.
  • style: Permite aplicar estilos CSS en línea para personalizar la apariencia de la opción.
  • data-*: Almacena datos personalizados en la opción, accesibles mediante JavaScript.

Ejemplo



Línea 10: `value="valor1"` Define el valor 'valor1' que se enviará al servidor cuando se seleccione esta opción.
Línea 11: `selected` Preselecciona esta opción, mostrándola como la opción seleccionada por defecto.

Etiqueta <textarea>

<textarea> </textarea>

Se utiliza para crear un campo de texto multilínea que permite a los usuarios ingresar y editar texto de forma más extensa que un campo de texto simple.

Atributos

  • id: Asigna un identificador único al `textarea`, útil para aplicarle estilos específicos o manipular su contenido con JavaScript.
  • name: Define el nombre del `textarea`, que se enviará junto con su contenido cuando se envíe el formulario.
  • rows: Establece el número de filas visibles del `textarea`.
  • cols: Establece el número de columnas visibles del `textarea`.
  • placeholder: Proporciona un texto de ayuda que aparece dentro del `textarea` cuando está vacío.
  • maxlength: Limita el número máximo de caracteres que se pueden ingresar en el `textarea`.
  • readonly: Hace que el `textarea` sea solo de lectura, impidiendo que el usuario modifique su contenido.
  • disabled: Deshabilita el `textarea`, impidiendo que el usuario lo use o modifique su contenido.
  • required: Indica que el `textarea` debe ser completado antes de enviar el formulario.
  • wrap: Define cómo se debe ajustar el texto dentro del `textarea`, con valores posibles como `soft` (ajuste automático) o `hard` (ajuste manual).
  • autofocus: Hace que el `textarea` reciba el foco automáticamente cuando se carga la página.
