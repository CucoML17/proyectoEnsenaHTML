Framework Materialize web

Materialize es un framework front-end basado en el diseño de interfaces llamado Material Design, que fue desarrollado por Google. Su propósito es facilitar la creación de sitios web y aplicaciones web atractivas y funcionales, siguiendo principios de diseño visual, usabilidad y accesibilidad.

Materialize ofrece una colección de componentes predefinidos que permiten a los desarrolladores construir rápidamente interfaces modernas y responsivas.

¿Qué incluye?

Materialize incluye una amplia gama de componentes como botones, tarjetas, menús de navegación, formularios y modales, que pueden ser fácilmente integrados en cualquier página web. Estos componentes siguen el estilo visual de Material Design, ofreciendo una apariencia coherente y profesional.

El framework está diseñado para ser totalmente responsivo, lo que significa que las páginas web creadas con Materialize se adaptan automáticamente a diferentes tamaños de pantalla, como móviles, tabletas y escritorios.

¿Cómo incluimos Materialize web en nuestras páginas?

  1. Primeramente, debemos diriginirnos a la página oficial de Materialize web para descargar los componentes CSS y Js (JavaScript) que Materialize web usará para implementar el framework: Descargalo aquí.
  2. Después de la descarga, se debe extraer los archivos en el directorio donde se encuentra su sitio web. El directorio se verá así.

    3. MyWebsite/
    |--css/
    | |--materialize.css
    |
    |--fonts/
    | |--roboto/
    |
    |--js/
    | |--materialize.js
    |
    |--index.html

  3. Notarás que hay dos conjuntos de archivos. El min significa que el archivo está "comprimido" para reducir los tiempos de carga. Estos archivos minificados generalmente se usan en producción, pero es mejor usar los archivos no minificados durante el desarrollo.
  4. A continuación sólo tienes que asegurarte de vincular los archivos correctamente en tu página web. Generalmente es aconsejable importar archivos javascript al final del cuerpo para reducir el tiempo de carga de la página. Badges

Las insignias pueden notificar que hay mensajes o notificaciones nuevos o no leídos.
Se debe agregar a la nueva clase en la insignia para darle el fondo.

Atributos

  • badge: Aplica el estilo básico de un Badge, que lo convierte en una pequeña etiqueta de notificación, generalmente posicionada a la derecha del elemento.
  • new: Añade un estilo destacado que indica un Badge "nuevo", resaltándolo visualmente para notificaciones recientes.
  • data-badge-caption: Personaliza la leyenda que acompaña al número en el Badge. Si no se especifica, solo aparecerá el número o el texto predeterminado.
  • positioning: Los Badges se alinean automáticamente a la derecha del elemento en el que se encuentran, como listas o enlaces.
  • color: Puedes personalizar el color del Badge utilizando clases de color como red, blue, green, etc.
  • size: Si necesitas cambiar el tamaño del Badge, esto se puede hacer ajustando los estilos CSS del texto o utilizando clases predefinidas de Materialize para fuentes.
  • hover: Los Badges en Materialize no tienen un comportamiento de hover por defecto, pero puedes añadir efectos con CSS personalizado para mejorar la interacción del usuario.
  • responsive: Los Badges son responsivos por defecto y se adaptan bien a diferentes tamaños de pantalla sin necesidad de ajustes adicionales.

Buttons

Hay 3 tipos de botones principales descritos en Material Design:

El botón elevado es un botón estándar que indica acciones y busca dar profundidad a una página mayoritariamente plana.

El botón de acción circular flotante está destinado a funciones muy importantes.

Los botones planos se suelen utilizar dentro de elementos que ya tienen profundidad como tarjetas o modales.

Atributos

  • raised: Utiliza la clase waves-effect waves-light btn para crear botones elevados que añaden profundidad a una página principalmente plana. Ejemplos de uso incluyen:
    • basic: <a class="waves-effect waves-light btn">button</a>
    • with icon left: <a class="waves-effect waves-light btn"><i class="material-icons left">cloud</i>button</a>
    • with icon right: <a class="waves-effect waves-light btn"><i class="material-icons right">cloud</i>button</a>
  • floating: Utiliza la clase btn-floating btn-large waves-effect waves-light para botones flotantes, ideales para funciones importantes. Ejemplo:
    • floating large: <a class="btn-floating btn-large waves-effect waves-light red"><i class="material-icons">add</i></a>
  • flat: Utiliza la clase waves-effect waves-teal btn-flat para botones planos que se usan para reducir el apilamiento excesivo, generalmente dentro de elementos con profundidad como tarjetas o modales. Ejemplo:
    • flat: <a class="waves-effect waves-teal btn-flat">Button</a>
  • submit: Para botones de envío de formularios, usa la etiqueta button con el atributo type="submit" en lugar de la etiqueta input. Ejemplo:
    • submit button: <button class="btn waves-effect waves-light" type="submit" name="action">Submit <i class="material-icons right">send</i></button>
  • large: Utiliza la clase btn-large para botones que necesitan más atención debido a su tamaño mayor. Ejemplos de uso incluyen:
    • large basic: <a class="waves-effect waves-light btn-large">Button</a>
    • large with icon left: <a class="waves-effect waves-light btn-large"><i class="material-icons left">cloud</i>button</a>
    • large with icon right: <a class="waves-effect waves-light btn-large"><i class="material-icons right">cloud</i>button</a>
  • small: Utiliza la clase btn-small para botones más pequeños, útiles en interfaces densas. Ejemplos de uso incluyen:
    • small basic: <a class="waves-effect waves-light btn-small">Button</a>
    • small with icon left: <a class="waves-effect waves-light btn-small"><i class="material-icons left">cloud</i>button</a>
    • small with icon right: <a class="waves-effect waves-light btn-small"><i class="material-icons right">cloud</i>button</a>
  • disabled: La clase disabled puede aplicarse a todos los tipos de botones para desactivarlos. Ejemplos incluyen:
    • large disabled: <a class="btn-large disabled">Button</a>
    • basic disabled: <a class="btn disabled">Button</a>
    • flat disabled: <a class="btn-flat disabled">Button</a>
    • floating disabled: <a class="btn-floating disabled"><i class="material-icons">add</i></a>

Breadcrumbs

Las rutas de navegación son una buena forma de mostrar la ubicación actual en un documento extenso o hecho de múltiples subpáginas o incluso páginas.

Esto generalmente se usa cuando tienes varias capas de contenido.

Atributos

  • class="breadcrumb": Aplica el estilo de breadcrumb a un enlace, indicándolo como parte de la navegación jerárquica. Esto estiliza el enlace para que se muestre como un elemento de breadcrumb.
  • class="nav-wrapper": Se utiliza dentro del contenedor <nav> para envolver el contenido y asegurar que los elementos de breadcrumb se alineen correctamente y ocupen el espacio adecuado.

Cards

Las tarjetas son un medio conveniente para mostrar contenido compuesto por diferentes tipos de objetos. También son adecuados para presentar objetos similares cuyo tamaño o acciones admitidas pueden variar considerablemente, como fotografías con leyendas de longitud variable.

Atributos

  • class="card": Define el contenedor principal para una tarjeta. Aplica el estilo básico de tarjeta con sombras y bordes redondeados.
  • class="card-image": Se utiliza para agregar una imagen a la parte superior de la tarjeta. Generalmente, se coloca dentro del contenedor de la tarjeta.
  • class="card-content": Define el área de contenido de la tarjeta, donde puedes colocar texto y otros elementos.
  • class="card-action": Se utiliza para agregar acciones a la tarjeta, como enlaces o botones, generalmente colocados al final de la tarjeta.
  • class="card-title": Se usa para definir el título de la tarjeta dentro del área de contenido. Aplica un estilo específico para el texto del título.
  • class="card-panel": Aplica un estilo de panel a la tarjeta, con un fondo sólido y bordes redondeados, ofreciendo un estilo alternativo a las tarjetas.
  • class="card-reveal": Define una sección que se revela al hacer clic en un botón o enlace dentro de la tarjeta. Se usa para mostrar contenido adicional.
  • class="card-stacked": Se utiliza para apilar la tarjeta, asegurando que el área de contenido y las acciones se ajusten adecuadamente en dispositivos móviles.
  • class="card-image img": Cuando se agrega una imagen dentro del contenedor card-image, asegura que la imagen se ajuste correctamente dentro del área de la tarjeta.

Floating Action Button

Si utiliza para crear un botón de acción flotante fijo, el cual se usa para agregar varias acciones que aparecerán al pasar el mouse.

Atributos

  • class="btn-floating": Aplica el estilo flotante a un botón. Esto crea un botón circular con sombra, que flota sobre el contenido.
  • class="btn-large": Cambia el tamaño del botón flotante a uno grande. Útil cuando se desea un botón más destacado.
  • class="btn-small": Cambia el tamaño del botón flotante a uno pequeño, ideal para interfaces más densas o menos invasivas.
  • class="fixed-action-btn": Aplica un estilo específico para fijar el botón flotante en la pantalla. Generalmente se utiliza en la parte inferior derecha y se mantiene visible al hacer scroll.
  • class="click-to-toggle": Añade funcionalidad para que el FAB expanda los botones secundarios al hacer clic, en lugar de hacerlo automáticamente al pasar el cursor.
  • class="toolbar": Convierte el FAB en una barra de herramientas expansible al hacer clic, mostrando botones adicionales en línea horizontal.
  • data-position="left/right/top/bottom": Define la posición en la que se desplegarán los botones secundarios del FAB. Los valores pueden ser left, right, top, o bottom.
  • data-tooltip: Asigna un tooltip al botón flotante, que se muestra al pasar el cursor sobre el botón.
  • href="#": Asigna una URL al FAB, permitiendo que el botón flotante actúe como un enlace cuando se hace clic en él.
  • i class="material-icons": Se utiliza para colocar íconos dentro del FAB. Generalmente, se utiliza con la clase material-icons para añadir íconos predefinidos de Materialize.

Propiedades dinámicas

  • direction: (String) Define la dirección en la que el menú FAB se abre. Los valores posibles son 'top', 'right', 'bottom', y 'left'. El valor predeterminado es 'top'.
  • hoverEnabled: (Boolean) Si es true, el menú FAB se abrirá al pasar el cursor sobre el botón, en lugar de hacerlo al hacer clic. El valor predeterminado es true.
  • toolbarEnabled: (Boolean) Si es true, habilita la transición del FAB a una barra de herramientas al hacer clic. El valor predeterminado es false.

Métodos

  • .open(): Abre el menú del FAB.
  • .close(): Cierra el menú del FAB.
  • .destroy(): Destruye la instancia del plugin y elimina cualquier configuración asociada.

Propiedades JS

  • el: (Element) El elemento DOM con el que se inicializó el plugin.
  • options: (Object) Las opciones con las que se inicializó la instancia.
  • isOpen: (Boolean) Describe el estado de apertura/cierre del FAB. Si está abierto, es true; de lo contrario, es false.

Navbar

La barra de navegación está contenida completamente en una etiqueta de navegación HTML5. Dentro de un div contenedor, hay 2 partes principales de la barra de navegación. Un logotipo o enlace de marca y los enlaces de navegación, el cual puede alinear estos enlaces a la izquierda o a la derecha.

Atributos

  • class="navbar": Aplica el estilo básico para la barra de navegación. Define el contenedor principal del navbar.
  • class="nav-wrapper": Envuelve el contenido de la barra de navegación, asegurando que el contenido esté alineado y estructurado correctamente.
  • class="brand-logo": Se utiliza para estilizar el logotipo o el nombre de la marca en la barra de navegación. Generalmente se encuentra en el lado izquierdo del navbar.
  • class="left": Alínea el contenido a la izquierda dentro del navbar. Usado comúnmente para el logotipo o enlaces de navegación.
  • class="right": Alínea el contenido a la derecha dentro del navbar. Usado para los enlaces de navegación o botones adicionales.
  • class="nav-item": Aplica el estilo a cada elemento de navegación dentro del navbar, como enlaces o botones.
  • class="sidenav": Se utiliza para crear un menú lateral que se despliega desde el borde de la pantalla, a menudo como una alternativa al navbar en pantallas más pequeñas.
  • class="sidenav-trigger": Añade un botón o enlace que activa el menú lateral (sidenav) en dispositivos móviles o cuando el navbar está colapsado.
  • data-target: Define el identificador del menú lateral que se debe abrir cuando se activa el botón o enlace con class="sidenav-trigger". Se usa para vincular el trigger con el sidenav correspondiente.
  • class="nav-mobile": Define los estilos específicos para el navbar en dispositivos móviles, a menudo mostrando un menú desplegable o un icono de menú.
  • class="transparent": Hace que el navbar sea transparente, permitiendo que el fondo del sitio web se vea a través del navbar.
  • class="fixed": Fija el navbar en la parte superior de la página, manteniéndolo visible mientras se desplaza por la página.
  • class="scrollspy": Activa el seguimiento del desplazamiento para resaltar el elemento de navegación activo en función del scroll de la página.

Pagination

Añadir enlaces de paginación ayuda a dividir su contenido extenso en bloques más cortos y más fáciles de entender; así como los resultados de búsqueda de múltiples navegadores o páginas que guardan grandes contenido de información como libros, acervos o enciclopedias.

Atributos

  • class="pagination": Aplica el estilo básico para la paginación. Define el contenedor principal de los elementos de paginación.
  • class="page-item": Define cada ítem individual en la paginación, como números de página y enlaces de navegación.
  • class="page-link": Estiliza los enlaces dentro de cada ítem de la paginación, proporcionando un estilo adecuado para los números de página y los controles de navegación.
  • class="disabled": Aplica un estilo para deshabilitar los ítems de la paginación, como los controles de navegación que no están disponibles en la página actual.
  • class="active": Resalta el ítem de la paginación que representa la página actualmente activa, aplicando un estilo diferente para que el usuario sepa en qué página se encuentra.
  • class="previous": Estiliza el enlace para ir a la página anterior en la paginación.
  • class="next": Estiliza el enlace para ir a la siguiente página en la paginación.
  • class="center-align": Centra la paginación horizontalmente en su contenedor, proporcionando un diseño más equilibrado.
  • class="left-align": Alinea la paginación a la izquierda dentro de su contenedor.
  • class="right-align": Alinea la paginación a la derecha dentro de su contenedor.

Preloader

El preloader es un componente que muestra una animación de carga mientras se está cargando contenido. Esto es útil para mejorar la experiencia del usuario, proporcionando una indicación visual de que se está procesando algo en el fondo.

Atributos

  • class="preloader-wrapper": Define el contenedor principal del preloader. Se utiliza para envolver los elementos del preloader y aplicar estilos generales.
  • class="active": Activa el preloader. Debe añadirse al contenedor del preloader para que sea visible y funcione.
  • class="spinner": Define el estilo del spinner (rueda giratoria) que se utiliza como preloader. Existen varios tipos de spinners que puedes usar.
  • class="spinner-layer": Crea las diferentes capas del spinner para proporcionar el efecto de rotación. Cada capa puede tener su propio color y estilo.
  • class="circle-clipper": Define un clipper circular que se utiliza para crear el efecto de rotación del spinner.
  • class="left": Alinea la capa del spinner a la izquierda dentro del contenedor del preloader.
  • class="right": Alinea la capa del spinner a la derecha dentro del contenedor del preloader.
  • class="gap-patch": Define un parche de espacio entre las capas del spinner para ajustar la apariencia de la rotación.
  • class="spinner-small": Ajusta el tamaño del preloader a un tamaño más pequeño, adecuado para lugares con menos espacio.
  • class="spinner-large": Ajusta el tamaño del preloader a un tamaño más grande, adecuado para ser más visible en espacios más amplios.
  • class="hidden": Oculta el preloader, útil cuando se necesita mostrar u ocultar el preloader dinámicamente mediante JavaScript.
